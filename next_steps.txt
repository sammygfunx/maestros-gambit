PASS 2 — PGN EXPORT/IMPORT + PUZZLE MODE  (Opus, high — batched #6+#7)
==================================================================
WHY: both are "chess credibility" features that reuse the existing engine, so they
batch cleanly. PGN is what chess players expect for saving/sharing/analyzing games;
Puzzle mode is near-free content that shows off the battle scenes.

WHAT PGN IS: Portable Game Notation — the universal plain-text chess format. A
header block of tag pairs (Event/Site/Date/White/Black/Result) followed by the
numbered move list in algebraic notation, e.g.:
   [White "Ivory Philharmonic"] [Black "Maestro Magnus"] [Result "0-1"]
   1. e4 e5 2. Nf3 Nc6 ... 0-1
The engine already stores the moves as SAN in game.sanHistory, so EXPORT is easy;
IMPORT means parsing that movetext back into engine moves. PGN lets a player save a
game, share it, or paste it into Lichess/chess.com for analysis.

----------------------------------------------------------------------------

add the new test (below), update PROJECT_STATE.txt + README.md, then commit & push.

This pass does TWO related chess features:

(A) PGN EXPORT/IMPORT.
 - Put the logic in a new js/pgn.js (MG.PGN), DOM-free + require()-able under Node.
 - EXPORT: build a valid PGN string from a finished/ongoing game — Seven-Tag-Roster

   result token. Wire a "Copy PGN" + "Download .pgn" action onto the game-over card
