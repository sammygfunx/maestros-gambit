V1.5 CHANGES — ONLINE PLAY (verified: chess 26/26, relay smoke OK, net e2e
  test OK — two clients mate in sync via a live relay; lobby screenshot +
  zero-overflow probe; capture-pipeline regression screenshot)
  - ARCHITECTURE: the game stays pure HTML/JS/Canvas with NO dependencies. Online
    is "mirror-and-relay": BOTH browsers run the same deterministic js/chess.js;
    a tiny stateless relay only forwards each move + a few control messages. No
    server-side chess, no accounts, no DB. Turn-based, so no rollback/prediction.
  - RELAY (server/relay/, standalone Node + `ws`, adapted from the Key Spar relay):
    hardened room-code WebSocket relay. 5-letter codes, 2 players/room, in-memory,
    heartbeats, idle/TTL cleanup, origin allow-list, /healthz, message validation.
    Env prefix MG_RELAY_*. GAME_CHANNELS reduced to two: 'move' {from,to,promo}
    and 'control' {action: resign|rematch-request}. Deploy free on Render (see
    server/relay/README.md). `npm run smoke` self-tests; tests/test_net.js drives
    two real clients through it to a synced checkmate. node_modules/ws is vendored
    so smoke/e2e run offline.
  - CLIENT (js/net.js — MG.Net): WebSocket glue. host(side)/join(code), auto
    start-match on pairing (host resolves the coin toss + tells the joiner who is
    Ivory), sendMove/sendControl, requestRematch, leave(). DEFAULT_RELAY_URL const
    at the top: set it to your wss:// URL for one-click Host/Join with nothing to
    type; otherwise the lobby's Server field is used (persisted as settings.relayUrl).
    Handlers (onStatus/onRoomCreated/onStartMatch/onMove/onControl/onPeerLeft/
    onError) are wired in main.js. Requires wss:// when the page is https.
  - LOBBY (index.html #screen-online, ui.js, style.css): setup screen gains a third
    mode 'online' (seg-mode). UI.applyMode(v) now drives the setup groups: difficulty
    is CPU-only, the side selector doubles as the host's preferred colour for online,
    and the start button reads "Go Online →" and opens the lobby instead of starting.
    Lobby = two cards (Host → reveals room code w/ Copy; Join → 5-letter input),
    a collapsible Server settings (relay URL; auto-opens when no default/saved URL),
    a live status line, Back. UI.online.{setStatus,showCode,reenable} are the
    main→lobby update surface. Velvet/gold styling; responsive block stacks the
    cards on phones. Dev hook: ?screen=online opens the lobby directly.
  - PIPELINE WIRING (main.js): session.mode 'online' joins 'cpu'/'2p'.
      * startGame: humanColor = assigned colour; session.hostColor tracked (who
        drives rematches); Undo button hidden (no free takebacks online); HUD names
        show "(You)/(Opponent)". startOnlineGame(myColor) is the entry from onStartMatch.
      * commitLocalMove(m): relays the move (MG.Net.sendMove) then plays it locally;
        onPointer's two commit points now call it. isHumanTurn() online = your colour
        AND MG.Net.paired; isCpuTurn() stays cpu-only so no AI fires.
      * Inbound moves: applyRemoteMove → remoteQueue → drainRemote() reconstructs the
        move from legalMoves() (same from/to/promo match the e2e test uses) and plays
        it when nothing else is animating (afterMoveResolve drains the queue). Robust
        if your battle scene is still playing when the opponent's reply arrives.
      * Checkmate/stalemate/draw need NO network message — both engines reach them
        independently. Only resign (control 'resign') and rematch are messaged.
      * resign online: ends locally + sends control; the peer ends as winner.
        Rematch: host drives (startHostRematch swaps colours via startMatchAsHost);
        the joiner sends 'rematch-request'. onPeerLeft ends the match with an
        "Opponent Left" card. quitToMenu calls MG.Net.leave() (peer sees you go).
        endGame plays fanfare/dirge per the local player's result for online too.
  - NOT DONE (deliberate, test-scope): no accounts/matchmaking/reconnect-resume/
    spectators/anti-cheat; in-memory rooms vanish on relay restart; one relay
    process only. Move integrity relies on both clients running the honest engine.
  - REAL-DEVICE / TWO-NETWORK CHECK still pending: only local relay + headless were
    exercised here. Deploy the relay (Render) and play from two networks before
    treating online as fully validated; confirm wss:// on an https-hosted build and
    Render free-tier cold-start (open /healthz first).

V1.3 CHANGES (all verified: 26/26 tests, soak 88/88, audio paths smoke-tested)
  - BUNDLED SOUNDTRACK (audio.js): a step-sequencer plays original, fully
    synthesized looping tracks through the existing instrument voices (NO
    samples/files/deps). TRACKS[] holds note-data: each track is {bpm, div
    (steps/beat), steps, parts:[{voice, vol, durBeats, seq}]}. seq is a sparse
    per-step row (helper row(len,[[step,val],...])); a cell is a note name
    ('A4','Bb2'), a chord (array of names), a number (block pitch), or null.
    noteFreq() converts note names -> Hz. A lookahead scheduler (_scheduler,
    25ms setInterval, 0.25s window) schedules each step via _playNote, which
    dispatches to pluck/bow/reed/piano/block etc. Three tracks ship:
    "Overture in Amber" (Am-F-C-G), "Clarinet Nocturne" (Dm-Bb-F-C),
    "Pizzicato Pavane" (C-G-Am-F).
  - MUSIC BUS (audio.js): new musicGain node parallel to sfxGain. A transient
    this._route field, set only around a synth voice call, makes the three
    output primitives (osc/noise/piano) connect to musicGain instead of
    sfxGain — so the synth music rides the Music Volume slider independently
    of SFX, reusing the SAME voice code. setMusicVol drives both musicEl and
    musicGain.
  - PER-CHARACTER STINGERS (audio.js stinger(type)): a short signature phrase
    per musician on its own voice (P pizz triad, N bowed fifth, B reed turn,
    R timpani+snare flam+cymbal, Q rolled piano arpeggio, K brass call).
    Played on piece selection (select() now delegates to stinger()) AND as the
    battle prelude (main.js runBattle calls Audio.stinger(attacker.t)).
  - SOUNDTRACK OPTIONS (ui.js + index.html): Options gains a Music on/off seg
    (#seg-music) and a Soundtrack track selector (#seg-track, Amber/Nocturne/
    Pavane); both persist in mg_prefs (settings.musicOn / settings.track).
    Toggling on or switching track live-previews. startGame plays the board
    music; quitToMenu stops it.
  - BRING-YOUR-OWN MUSIC kept and improved: a Clear button (#btn-clear-music)
    stops playback, nulls musicEl, revokes the object URL, and resets the
    filename label, reverting to the bundled track. A loaded user file still
    overrides the synth track while present. Engine API: startBoardMusic/
    stopBoardMusic (playMusic/stopMusic kept as aliases), loadMusicFile/
    clearMusicFile, setMusicOn/setMusicTrack.

WHAT THIS IS
  Battle Chess–style game, orchestra theme. HTML5/Canvas, zero dependencies,
  runs by opening index.html. Name: "Maestro's Gambit". Teams: Ivory
  Philharmonic (white) vs Obsidian Philharmonic (black).
  Pieces: P=Violinist N=Cellist B=Clarinetist R=Percussionist Q=Pianist
  K=Conductor.

    grandFinale(), one of three random sends-off (curtain+crash / bouquet-note

    each gained a 2nd take ending on the FINALE() beat.

    perft(4)=197281 exact. Run: node tests/test_chess.js
  - AI: negamax+alpha-beta+quiescence (fail-soft — important, a fail-hard
    quiescence bug was fixed where narrowed root windows returned beta as

    Student(d2+jitter+blunders) / Performer(d3) / Virtuoso(d4, ~600ms/move).
  - 76 battle choreographies in js/battle.js: 36 primary (CHOREO) + 40 alts
    (ALT), keyed 'X>Y' (attacker>defender), chosen at random per battle.
    '*>K' entries are the checkmate finales (grandFinale: curtain + applause +
    confetti, 3 random variants). Fallback chain exists for safety. Plus the
    set-piece scenes: castling (2), en passant (2), promotion (4, one per
    promote type), stalemate (2), draw (2) — 88 sequences total, all soaked.
  - Sprites are 100% procedural pixel art: js/sprites.js. Each character is
    a parametric rig (POSES[type][action](k,t,P) mutates a pose; painters
    draw to a 110x110 lowres buffer upscaled with smoothing off).
    Actions: idle walk ride windup strike special hit dodge block taunt
    die dead win cheer.
  - SFX all synthesized in js/audio.js (no samples). User loads own music
    file via Options (plays looped; cannot persist across sessions because
    browsers can't re-read local files — a bundled-soundtrack build would).
  - Menus/HUD/promotion/game-over all DOM (index.html + css/style.css).
  - Modes: vs CPU (3 levels, pick side or coin toss), 2P local, and 2P online
    (room-code relay; see V1.5 CHANGES + server/relay/). Battles toggleable
    in-game (per-player when online); Esc/click skips a battle; speed setting
    Normal/Brisk/Presto; prefs persist (localStorage key 'mg_prefs', now incl.
    settings.relayUrl).

ARCHITECTURE CHEAT SHEET
  Namespace: globalThis.MG. Script order matters (see index.html).
  main.js App: state machine 'menu'|'board'|'battle', move pipeline:
    executeMove -> snapshot -> game.move -> board.beginMoveAnim
    -> onArrive -> (battle if capture & enabled) -> afterMove
    -> status checks (mate => optional finale battle => endGame)
    -> cpuMove if needed.
  Board is authoritative immediately (engine state applied before anim);
  board view renders from a snapshot during animation.
  Captured trays tracked in App.capturedByWhite/Black (undo-aware).
